# AGENTS.md

这份文档给 AI 编码助手阅读，用来维护「很大声周刊」仓库。改动时优先遵守这里的约定，并尽量保持现有写法和排版。

## 仓库结构

- `README.md` 是周刊总索引。
- `docs/vol.NNN.md` 是每一期周刊正文。
- 新一期通常只需要新增 `docs/vol.NNN.md`，并把这一期补到 `README.md` 顶部对应月份。
- 不要批量重排历史条目，不要顺手修旧文格式，除非用户明确要求。

## 新增一期后的 README 更新流程

1. 读取最新一期正文，例如 `docs/vol.164.md`。
2. 从正文提取目录条目：
   - 一级标题 `#` 作为 README 的主条目。
   - 二级标题 `##` 可以作为上一条的缩进子条目。
   - 正文标题 `# 很大声周刊-vol.NNN` 不写进目录。
3. 在 `README.md` 顶部当前月份分组下插入新一期，放在上一期之前。
4. 使用现有格式：

```md
* **[vol.NNN](docs/vol.NNN.md)**
    <small>
  * 一级标题
  * 另一个一级标题
    * 二级标题
  * 小白兔白又白
  * 歌曲名 - 艺人
    </small>
```

5. 如果 README 顶部没有对应月份，就按现有结构新增月份标题；如果年份分组明显缺失或不一致，先保持当前文件的局部风格，必要时向用户确认。

## 写作和格式约定

- 保持中文标点和原文标题，不要自动润色正文。
- 图片链接、外链、歌曲名、人名保持原样。
- README 目录只做摘要，不要把正文段落复制进去。
- 缩进沿用现有 README：`<small>` 前通常是 4 个空格，目录项前通常是 2 个空格。
- 文件名使用现有格式：`vol.164.md`，不要改成补零或其他命名。

## Git 注意事项

- 修改前先看 `git status -sb`，不要覆盖用户未提交内容。
- 如果只是在更新索引，通常只改 `README.md`。
- 新增一期正文是用户写好的内容时，不要擅自改正文。
- 提交或推送只有在用户明确要求时才做。
