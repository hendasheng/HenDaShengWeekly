# Ableton Extensions - Ableton 基于 JS 的自定义扩展
![img](https://hendasheng-web.oss-cn-beijing.aliyuncs.com/Weekly/vol.167/ableton-extensions-v4-b-3000x2200.png)

[Ableton Extensions](https://www.ableton.com/en/live/extensions) 是 Ableton 在 Live 12.4.5 中引入的新工具。它可以理解为一种新的扩展方式：通过 Ableton Extensions SDK，用户可以为 Live 编写自己的小工具，让 Live 不再只是一个被操作的软件，而是可以被进一步定制和自动化的创作环境。

相比 [Max for Live](https://www.ableton.com/en/live/max-for-live/) ，[Ableton Extensions](https://www.ableton.com/en/live/extensions)  更偏向声音生成、乐器设计和复杂信号处理，Ableton Extensions 更像是把 JavaScript 工具接入 Live Set 本身。它可以直接作用于工程结构、音乐数据和工作流，让开发者用代码去整理、生成或改写 Live 里的内容。

# The Book of Shaders - Shaders 入门指南
![img](https://hendasheng-web.oss-cn-beijing.aliyuncs.com/Weekly/vol.167/shader.png)

> 如果你曾经有用计算机绘图的经验，你就知道在这个过程中你需要画一个圆，然后一个长方形，一条线，一些三角形……直到画出你想要的图像。这个过程很像用手写一封信或一本书 —— 都是一系列的指令，需要你一件一件完成。
> 
>[Shaders](https://thebookofshaders.com/01/?lan=ch)  也是一系列的指令，但是这些指令会对屏幕上的每个像素同时下达。也就是说，你的代码必须根据像素在屏幕上的不同位置执行不同的操作。就像活字印刷，你的程序就像一个 function（函数），输入位置信息，输出颜色信息，当它编译完之后会以相当快的速度运行。

# Ableton Operator 合成器二十周年回顾
![img](https://hendasheng-web.oss-cn-beijing.aliyuncs.com/Weekly/vol.167/operator-main.jpg)

[Operator](https://www.ableton.com/en/packs/operator/) 在 Ableton Live 中的重要性不言而喻，作者 [Robert Henke](https://roberthenke.com/interviews/bio.html) 也是 Ableton Live 的主要创造者之一，在 [Operator 20 周年（2024）](https://roberthenke.com/technology/operator.html) 之际发布了这篇文章，介绍了关于 Operator 的背景和一些技巧。

![img](https://hendasheng-web.oss-cn-beijing.aliyuncs.com/Weekly/vol.167/Robert%20Henke-Engineering.png)

作者的个人网站非常值得研究，尤其是关于 [工程](https://roberthenke.com/technology/index.html) 部分，页面里可以看到大量 Ableton Live 乐器、效果器与工具的整理，从 Operator、Simpler 到 Beat Repeat、Grain Delay、Erosion 等，很多平时直接拿来用的 Live 内置设备，背后都和 Henke 的声音审美与工程思路有关。
